# 🎮 Game Challenges Hub

A responsive, dependency-free web app of brain-teasing game challenges. The
flagship challenge is a fully featured **Tower of Hanoi** puzzle.

> **▶️ Live demo:** https://SkillsCampMbz2026.github.io/game-challenges-hub/

## Challenges

The [hub menu](index.html) lists all challenges. Currently playable: **Tower of
Hanoi** and **Lights Out**.

### 🗼 Tower of Hanoi

Move the entire stack of disks from the **Start** tower to the **Goal** tower
using the fewest moves possible — never placing a larger disk on a smaller one.

**Features**

- **Three towers** with disks you can **drag _or_ click** to move.
- **Valid-move checking** — illegal moves are rejected with visual feedback.
- **Move counter** and a **minimum-moves** display (`2ⁿ − 1`).
- **Difficulty selection** by number of disks (3–8).
- **Restart** button and clear **instructions / objective**.
- **Auto-solve** button that animates the optimal solution (not saved to the
  leaderboard); click again to stop.
- **Automatic victory detection** with a summary of your run.
- **Live timer** for completion time.
- A **localStorage leaderboard** per difficulty, saving player name, disk
  count, moves, and completion time (ranked by moves, ties broken by time).

### 💡 Lights Out

Turn off **every light** on a 5×5 grid. Clicking a cell toggles it and its four
orthogonal neighbours (up/down/left/right).

**Features**

- **5×5 grid** of clickable lights with a cross-toggle rule.
- **Move counter**, live **timer**, and a **lights-on** readout.
- **Randomised but always-solvable** starting boards — generated by scrambling
  a blank board, so a solution is guaranteed to exist.
- **Difficulty options** (Easy → Expert) that target boards whose true shortest
  solution needs progressively more moves.
- **Minimum-moves display** computed exactly by a GF(2) linear-algebra solver.
- **Hint** button that reveals one move from an optimal solution.
- **Restart** button, clear **instructions / objective**, and **automatic
  victory detection**.
- A **localStorage leaderboard** per difficulty, saving player name, difficulty,
  moves, and completion time.

> The solver treats a press as a vector over GF(2) and solves `A·x = b` by
> Gaussian elimination, enumerating the board's null-space to return the
> fewest-presses solution. (It correctly finds the classic result that the
> all-lit 5×5 board needs exactly **15** presses.)

## How to play — Tower of Hanoi

- **Click** a tower to lift its top disk, then click another tower to drop it.
- Or **drag** the top disk onto another tower.
- A disk may never rest on a smaller disk; only the top disk of a tower moves.
- Rebuild the full stack on the **Goal** tower to win. Match the minimum
  move count for a ✨ **perfect** run.

## Project structure

```
index.html                 # Game Challenges hub menu
tower-of-hanoi.html        # The Tower of Hanoi game
lights-out.html            # The Lights Out game
css/styles.css             # Shared responsive styles (dark theme)
js/game-core.js            # Hanoi logic (rules, solver) — shared by app & tests
js/hanoi-ui.js             # Hanoi UI: drag/click, timer, auto-solve, leaderboard
js/lights-out-core.js      # Lights Out logic + GF(2) solver — shared by app & tests
js/lights-out-ui.js        # Lights Out UI: grid, hint, timer, leaderboard
test/solve.test.js         # Hanoi logic test (beats every difficulty optimally)
test/ui.test.js            # Hanoi UI test (jsdom)
test/lights-out.test.js    # Lights Out solver test (incl. the 15-move result)
test/lights-out-ui.test.js # Lights Out UI test (jsdom): solves via clicks
```

The game logic lives in `js/game-core.js` with **no DOM dependencies**, so the
exact same rules that run in the browser are unit-tested in Node.

## Running locally

No build step. Open `index.html` in a browser, or serve the folder:

```bash
npx serve .        # or: python -m http.server
```

## Tests

```bash
npm install        # for the jsdom-based UI test
npm test           # solves & beats disks 3–8 in the minimum moves
npm run test:ui    # plays the real UI in jsdom and beats a 3-disk game
```

Both suites pass — the puzzle is verifiably beatable (and beaten) at every
difficulty in the optimal number of moves.
